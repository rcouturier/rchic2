RCHIC2 Statistical Implicative Analysis

User guide for the RCHIC2 application, a tool for Statistical Implicative Analysis (SIA) designed to study relationships between binary or categorical variables.

1. Quick Start

  1. Launch RCHIC2.
  2. Click Load a CSV file and select your data file.
  3. Verify that your variables appear in the right-hand panel.
  4. Choose an analysis type: Implicative Graph, Similarity Tree, or Hierarchy Tree.
  5. Adjust options if needed, then click Compute.
  6. Explore the result in the Visualization tab and the details in the Computation Data tab.

3. Application Interface

Sidebar (left)

  • Load a CSV file: opens the file selector.
  • Information: file name, number of observations and variables.
  • Compute: runs the selected analysis.
  • Export: saves the image and data.
  • Analysis type: choice between the three analysis types.
  • Options: settings specific to each analysis type.

Variables Panel (right)

  • List of all variables with checkboxes.
  • All / None buttons for quick selection.
  • Only checked variables are included in the computation.

Main Area

  • Visualization tab: displays the graph or tree.
  • Computation Data tab: displays the console with detailed results.

Language Selector

In the top-right corner, choose between FR (French), EN (English), or PT (Portuguese). The language affects the interface, console messages, and CSV export format.

4. Implicative Graph

The implicative graph represents the statistical implication relationships between variables. An arc A → B means: "when A is true, B also tends to be true".

Computing Modes

ModeDescription
ClassicStandard implication index.
Classic + ConfidenceCombines the implication index with rule confidence.
ImplifianceImplifiance index, specific to SIA.
EntropicEntropy-based index (information theory).

Thresholds and Colors

Four independent thresholds (50 to 100) control which edges are displayed:

  • Each threshold has an enable checkbox, a slider, and a color picker.
  • An edge is displayed if its implication value exceeds at least one active threshold.
  • The edge color corresponds to the highest threshold it exceeds.

Additional Options

  • Complete graph: shows implications in both directions (A→B and B→A).
  • Show confidence: displays the confidence value on each edge.

Interacting with the Graph

  • Click on an edge: displays rule details (implication, confidence, counter-examples).
  • Click on a node: displays the variable name and its occurrences.
  • Right-click on a node: computes contributions and/or typicalities of supplementary variables from that node (see section 7).

5. Similarity Tree

The similarity tree groups variables by their degree of resemblance. The more similar two variables are in their responses, the closer they appear in the tree.

Options

  • Show similarity: displays the similarity value (0-1) at each merge level.
  • Contributions / Typicalities: enables computation with supplementary variables (see section 7).

Reading the Tree

  • Variables are displayed at the top (leaves).
  • Horizontal lines indicate merges between groups.
  • Merges in red are statistically significant.
  • Similarity values appear above each merge.

6. Hierarchy Tree

The hierarchy tree groups variables into oriented classes, taking the direction of implication into account. Cohesion measures the strength of the bond within each class.

Computing Modes

ModeDescription
ClassicStandard cohesion.
Classic + ConfidenceConfidence-weighted cohesion.
ImplifianceImplifiance-based cohesion.

Options

  • Show cohesion: displays the cohesion value at each level.
  • Contributions / Typicalities: enables computation with supplementary variables (see section 7).

7. Contributions and Typicalities

These analyses characterize the classes or paths identified by the analysis using supplementary variables (columns with the .s suffix).

Prerequisite: Your CSV file must contain at least one supplementary variable (.s suffix) for these computations to be available.

Contribution

Measures how much each supplementary variable contributes to the formation of a class or path. A low risk indicates a strong contribution.

Typicality

Measures how representative (typical) each supplementary variable is of a class or path. A low risk indicates strong typicality.

How to Enable

Analysis TypeMethod
Implicative GraphRight-click on a node → context menu with 3 choices: Contributions, Typicalities, or both.
Similarity TreeCheck Contributions and/or Typicalities in the "Supplementary variables" section of the options, before computing.
Hierarchy TreeSame as the similarity tree.

Reading the Results

Results appear in the Computation Data tab. For each class or path:

  • Each supplementary variable is listed with its risk (probability).
  • The variable with the lowest risk is highlighted (>>).
  • A risk < 0.05 is generally considered significant.

8. Exporting Results

The Export button saves both the image and the data:

CSV format: The CSV file format depends on the selected language. In French and Portuguese, the separator is semicolon (;) with comma as decimal separator. In English, it is comma (,) with period as decimal separator.

9. Computation Console

The Computation Data tab displays all numerical results:

  • Computation parameters (mode, threshold, enabled options).
  • Number of variables, rules, nodes, edges.
  • Classification details (for trees).
  • Contributions and typicalities (if enabled).
  • Rules table (for the implicative graph).

Use Refresh to reload and Clear to empty the console.
